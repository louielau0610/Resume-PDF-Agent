# Product Spec v0

## 产品问题

学生、实习申请者和早期职业候选人通常有很多零散经历，但很难把这些信息整理成清晰、真实、适合 ATS 的 PDF 简历。常见问题包括：

- 经历描述过长或重点不清。
- 项目、课程、技能和目标岗位之间缺少关联。
- 简历内容容易出现夸大、空泛或无法证明的表述。
- 模板选择和最终 PDF 输出流程不稳定。

`resume_pdf_agent` 的目标是建立一个可控的简历生成工作流，把用户职业画像转化为结构化简历内容，并在后续阶段生成 PDF 简历。

## 目标用户

- 在校学生。
- 实习申请者。
- 早期职业候选人。
- 需要英文或中文简历基础版本的求职者。

## 核心使用场景

- 用户输入教育背景、课程、项目、实习、科研、技能和奖项。
- 系统将零散输入整理成结构化简历内容。
- 系统根据目标岗位和背景判断简历类型。
- 系统在不编造事实的前提下改写 bullet points。
- 系统检查是否存在缺少证据的强表述或虚构指标。
- 系统匹配内部模板并生成 PDF。

## 用户输入字段

计划支持的输入字段包括：

- 专业。
- GPA。
- 核心课程。
- 目标岗位。
- 教育背景。
- 项目经历。
- 实习经历。
- 科研经历。
- 技能。
- 奖项。
- 语言能力。
- 用户提供的证明材料或补充说明。

## 预期输出

v0 的目标输出是 PDF 简历。系统应提醒用户：当前版本只生成 PDF。如需 Word、JPG 或 PNG，可使用外部 AI 工具或 PDF 转换工具。

## Non-Goals

- 不实现真实 LLM 调用。
- 不接入 OpenAI、Anthropic 或其他模型 API。
- 不联网搜索简历模板。
- 不抓取互联网简历模板。
- 不实现 PDF 生成前的视觉设计系统。
- 不实现前端 UI。
- 不实现 Word、JPG 或 PNG 导出。
- 不编造用户未提供证据的成果、指标或经历。

## 风险

- 用户输入不足导致简历内容空泛。
- 用户提供的经历真实性难以完全验证。
- 过度依赖 LLM 可能导致虚构指标或夸大成果。
- 模板与内容长度不匹配可能导致版式问题。
- 多格式导出会增加 v0 范围和质量风险。

## MVP Scope

MVP 应聚焦于：

- 收集和验证基础用户画像。
- 生成结构化简历内容。
- 分类简历类型。
- 进行真实性检查。
- 匹配内部模板。
- 输出 PDF。
- 明确提示 PDF-only 限制。

M0 只完成项目初始化，不实现上述业务逻辑。
